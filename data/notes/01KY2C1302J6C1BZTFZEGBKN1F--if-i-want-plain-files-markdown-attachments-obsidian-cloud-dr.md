If I want my second brain as plain files (Markdown notes + attachments) the most practical options are Obsidian with a cloud backend, or a self-hosted file-sync solution. Obsidian Sync (paid) offers tight integration with Obsidian vaults, automatic syncing of attachments and backlinks, and is simplest if I use Obsidian as the editor. Alternatively, I can store Obsidian vaults on iCloud Drive (best for Apple-first workflows), Dropbox, or OneDrive for broad OS support; these services provide standard file sync, version history, and easy local file access. If I prioritize maximum control and privacy, I can self-host Nextcloud on a VPS or NAS. Nextcloud gives file sync, calendar/contacts, and app integrations, but requires maintenance (updates, backups) and occasional troubleshooting. Trade-offs: file-based approaches maximize portability (I can open notes with any text editor and keep a folder-based archive), while app-based syncs sometimes use proprietary formats. If cross-platform support and the ability to export raw markdown are important, prefer Dropbox/OneDrive or self-hosted Nextcloud; if tight Obsidian features and convenience matter more, Obsidian Sync is worth the paid tier.
